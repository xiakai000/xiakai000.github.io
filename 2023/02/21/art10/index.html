HTTP协议总结

发表于2023-02-21|更新于2023-07-02
|字数总计:1k|阅读时长:3分钟|阅读量:

HTTP是什么？

HTTP是一种超文本传输协议，工作在应用层（OSI 七层模型，或TCP/IP五层（或四层）模型），我们平时打开一个网站, 就是通过 HTTP 协议来传输数据的。

当我们在浏览器输入网址时，浏览器就给服务器发送一个HTTP请求，服务器就返回了一个HTTP响应

HTTP 协议格式

HTTP 是一个文本格式的协议，可以通过抓包工具抓包来查看请求和响应的内容。

协议格式如下：请求行/状态行，header（报头），空行，body（请求/响应正文）

其中空行是为了标记header结束，与body分隔开，因为HTTP在传输层依靠TCP协议，而TCP是面向字节流的，没有空行容易出现“粘包”问题。

其中请求行是以method url version 的语句组成，其中method方法为get，post，put，delete，其中get为向服务器获取数据，而post为向服务器提交数据，这两种较为常见。

状态行是响应返回的状态，version 状态码 状态码描述。

200 OK 这是一个最常见的状态码, 表示访问成功.
404 Not Found 没有找到资源.
403 Forbidden，表示访问被拒绝. 有的页面通常需要用户具有一定的权限才能访问(登陆后才能访问). 如果用户没有登陆 直接访问, 就容易见到 403.
405 Method Not Allowed 对方服务器不支持HTTP 中的方法, 虽然有 GET, POST, PUT, DELETE 等. 但是对方的服务器不一定都支持所有的方法(或者不允许用户使用一些其他的方法).
500 Internal Server Error 服务器出现内部错误. 一般是服务器的代码执行过程中遇到了一些特殊情况(服务器异常崩溃)会产生这个 状态码.
504 Gateway Timeout 当服务器负载比较大的时候, 服务器处理单条请求的时候消耗的时间就会很长, 就可能会导致出现超时的 情况.
302 Move temporarily 临时重定向.
301 Moved Permanently 永久重定向. 当浏览器收到这种响应时, 后续的请求都会被自动改成新的地址
header部分上是以键值对的形式组成，例如：content-type：正文的类型，content-length:正文的长度等。每个键值对占一行. 键和值之间使用分号分割；

body中的内容格式和 header 中的 Content-Type 密切相关，

  1. application/x-www-form-urlencoded （特殊字符会经过urlencode处理）

  2. multipart/form-data （上传文件用的格式）

  3. application/json （json格式的正文，最为常见）

    构造HTTP请求的方式

    一种是通过form表单来构造请求，get或者post请求。

    type: 表示输入框的类型. text 表示文本, password 表示密码, submit 表示提交按钮. name: 表示构造出的 HTTP 请求的 query string 的 key. query string 的 value 就是输入框的用户 输入的内容. value: input 标签的值. 对于 type 为 submit 类型来说, value 就对应了按钮上显示的文本.

另一种是通过ajax的方法来构造HTTP请求。

  特点是可以不需要 刷新页面/页面跳转 就能进行数据传输.

  这种方式本质上是通过js的XMLHTTPRequest对象来提交数据，但是比较复杂。现在大多使用jQuery的方式来构造。

那么HTTPS是什么？为什么要加S呢？

HTTPS 也是一个应用层协议. 是在 HTTP 协议的基础上引入了一个加密层. 因为HTTP 协议内容都是按照文本的方式明文传输的. 所以就导致在传输过程中出现一些被篡改的情况.

S为SSL（Secure Sockets Layer）或TLS（Transport Layer Security）。
